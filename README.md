# 물류팀 내부 업무 도구

물류팀 전용 내부 운영 도구 모음. 다크 네이비 테마로 전 페이지 통일된 디자인을 제공합니다.

---

## 완료된 기능

### 1. 일일 업무 현황 (`index.html`)
- 날짜별 보고서 작성 및 로컬스토리지 저장
- 오전/오후 코스 선택, 포장 현황, 기타 업무, 특이사항 입력
- 텔레그램 자동/수동 전송
- 인쇄 기능

### 2. 업무 분장표 (`duty.html`)
- 대/중/소분류 기반 업무 담당자 배정 (주담당 ● / 부담당 ○)
- 클릭으로 담당 순환 변경, 변경사항 자동 저장
- 업무 추가/수정/삭제 모달
- 드래그앤드롭 행 순서 변경
- 중분류 고정 순서: 정기 반복 → 관리·운영 → 돌발·지원 → 프로젝트성
- 담당자 통계 바

### 3. 팀원 공유 (`share.html`)
- 팀원별 오늘 한 일 / 공유 업무 입력
- 날짜별 조회 및 저장 (Table API)
- 텔레그램 전송

### 4. 물류 업무 플로우차트 (`flow/flowchart.html`)
- 노드 기반 업무 플로우 시각화 및 편집
- 드래그/줌/이동 지원
- 저장 기능 (로컬스토리지)

### 5. 출고 집계 (`shipment.html`)
- 날짜·채널·시간대(오전/오후/종일)·건수·메모 입력 (빠른 입력 폼)
- 기간 필터 (오늘 / 이번 주 / 이번 달 / 사용자 지정)
- KPI 카드: 총 출고 건수, 오늘 출고, 일 평균, 채널 수
- 채널별 집계 바 차트 (비율·건수 표시)
- 날짜별 추이 차트 (전체 선 차트 / 채널별 누적 막대 차트)
- 상세 내역 테이블 (검색·채널·시간대 필터, 열 정렬)
- 채널 관리 모달 (추가/삭제)
- CSV 내보내기
- 데이터: localStorage (`shipment_records_v1`, `shipment_channels_v1`)

### 6. 매핑관리 / 출고집계 도구 (`aggregator.html`)
- **자동 컬럼 감지** (헤더 키워드 → 데이터 내용 추론 → 고정 G/H열 폴백) 후 자사몰+사방넷 집계
- **마스터 데이터** 관리: 본사 품목코드·품명·환산비율 등록 (직접입력 / 엑셀 가져오기 / 내보내기)
- **채널 SKU 매핑**: 채널별 SKU코드 + 상품명 이중 매칭 (SKU 우선 → 정규화 → 포함 매칭)
- **🆕 번들 상품 지원**: 1개 엑셀 상품명 → 여러 본사 품목 동시 출고 처리
  - 예: "하카H 매트블랙(캡슐5개 증정)" → 본체(×1) + 캡슐(×1) 각각 집계
  - 매핑 등록 시 **[+ 품목 추가]** 버튼으로 추가 품목 설정
  - 각 품목별 수량배수 독립 설정 가능
  - 저장된 매핑 테이블에 번들 배지·품목 목록 표시
- **수량배수 환산**: 주문수량 × 배수 = 본사 기준수량 자동 계산
  - 5개입→×1, 10개입→×2, 20개입→×4, 40개입→×8 (커스텀 설정 가능)
- **미매핑 항목 처리**: 집계 후 누락 상품을 모달에서 즉시 매핑 + 재집계
- **출력**: 클립보드 복사(구글 시트) / 엑셀(.xlsx) / 출고집계 페이지 자동 전송
- **20:00 자동초기화**: 업로드 파일 매일 20시 리셋, 매핑데이터는 유지
- 데이터: localStorage (`agg_channels_v3`, `agg_items_v3`, `agg_mapping_v3`)

#### 번들 매핑 데이터 구조 (`agg_mapping_v3`)
```json
{
  "id": "...",
  "channel_id": "채널ID",
  "sku": "채널고유SKU (선택)",
  "excel_name": "주문서 상품명",
  "items": [
    { "item_id": "본사품목ID_1", "ratio": 1 },
    { "item_id": "본사품목ID_2", "ratio": 1 }
  ]
}
```
※ 구형 `item_id + ratio` 단일 구조는 자동으로 `items` 배열로 마이그레이션

### 7. 팀원 관리 (공통 상단 바)
- 메인 페이지 상단에서 팀원 추가/수정/삭제
- 변경 시 업무분장표·팀원공유 자동 반영 (localStorage + 이벤트 버스)

## 데이터 구조

### localStorage 키

| 키 | 설명 |
|---|---|
| `shipment_records_v1` | 출고 집계 기록 배열 `[{id, date, channel, slot, count, memo}]` |
| `shipment_channels_v1` | 채널 목록 배열 `string[]` |
| `duty_rows_v2` | 업무 분장 행 배열 |

| `team_share` | 팀원 공유 기록 (Table API) |
